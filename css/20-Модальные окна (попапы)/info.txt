Вариант 1
Без использования JS, только на CSS
Используется способность псевдокласса :target

Недостаток:
Полосы прокрутки при открытии попапа при большом его содержимом не очень корректно работают.
Плюс закрытие попапа вне области окна делается ссылкой на всю область
----------------
Вариант 2
С использованием JS
Простой вариант.
Только открытие и только закрытие (закрытие происходит как по кнопке "X", так и при клике вне области попапа)
Вариант оставлен, чтобы понять саму суть работы JS по открытию и закрытию попапа без всяких ухищрений.
Недостаток:
При большом объеме контента попапа сбоку появляется второй скролл для самого попапа, а также остается основной скролл всей страницы. 
Выглядит не очень презентабельно.
Реализована возможность открытия второго попапа, который сам находится в попапе. При этом уже открытый попап закрывается.
----------------
Вариант 3
С использованием JS
Сложный вариант (это Вариант 2 с доработками).
Он дорабатывает проблемы со скроллами.
При открытии попапа, основной скролл страницы убирается.
В JS используется переменная const timeout = 800.
Значение 800 должно соответствовать количеству милисекунд, которое задается в CSS в методе transition для анимации плавного появления попапа.
В данном случае у нас свойство
transition: all 0.8s ease 0s;
Поэтому
timeout = 800

Вся работа скрипта очень подробно построчно объясняется в ролике автором
